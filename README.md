# Skill Creator (`skcr`)

`skcr` is a Go CLI for creating versioned AI agent skill structures and rendering agentic project and platform files.

```text
skcr  = init / scaffold / bake / validate / clean
skpm  = validate / version / package / publish / install / update / lock / verify
```

`skcr` creates and renders. `skpm` manages the skill lifecycle.

## Architecture

```text
skcr init
  → creates agentic.bake.yaml

skcr scaffold skill <name>
  → creates one canonical skill source skeleton from CLI flags

skcr scaffold skills
  → creates all canonical skill sources declared in agentic.bake.yaml

skcr bake <target>
  → creates declared skill sources and renders platform-specific project output

skpm
  → validates, versions, packages, publishes, installs, locks, updates, and verifies skills
```

### Conceptual model

```text
skills/<name>/                    ← canonical skill source (committed to Git)
  ├── SKILL.md
  ├── skill.yaml
  ├── VERSION
  ├── CHANGELOG.md
  ├── README.md
  ├── LICENSE
  └── tests/
      └── README.md

.agents/skills/<name>/SKILL.md   ← Codex platform output    (generated by skcr bake)
.claude/skills/<name>/SKILL.md   ← Claude Code platform output
.github/skills/<name>/SKILL.md   ← GitHub Copilot platform output
.agents/skills/<name>/SKILL.md   ← GitLab Duo platform output
```

The source of truth is `skills/<name>/`. Platform outputs are generated by `skcr bake` and tracked in `.agentic-template.lock`.

## Supported platforms

- `codex`
- `claude-code` (alias: `claude`)
- `gitlab-duo` (alias: `gitlab`, `duo`)
- `github-copilot` (alias: `copilot`, `github`)

## Prerequisites

- Go `>= 1.23`
- `make` (optional, for `make build` and `make install`)

## Installation

```bash
# From source
make install

# Directly with Go
go install ./cmd/skcr

# From GitHub
go install github.com/domehahn/skcr/cmd/skcr@latest
```

## Quick start

### Initialize a project

```bash
skcr init --target . --project-name MyProject --platform codex,gitlab-duo
skcr bake default --target . --write
skcr validate --target .
```

### Create skill sources and bake

```bash
# Edit agentic.bake.yaml and add entries to skill_sources.skills, then:
skcr scaffold skills --target .
skcr bake default --target . --write
```

### One-shot skill creation

```bash
skcr scaffold skill gitlab-policy-reviewer \
  --output-dir ./skills \
  --version 0.1.0 \
  --platform codex \
  --platform gitlab-duo \
  --description "Reviews GitLab security policies." \
  --owner platform-security \
  --license MIT
```

## Commands

| Command | Description |
| --- | --- |
| `skcr init` | Create `agentic.bake.yaml` |
| `skcr scaffold skill <name>` | Create one canonical skill skeleton from CLI flags |
| `skcr scaffold skills` | Create all canonical skill skeletons from `agentic.bake.yaml` |
| `skcr bake <target>` | Create skill sources and render platform-specific output |
| `skcr validate` | Validate configuration and generated state |
| `skcr clean` | Remove skcr-managed files listed in `.agentic-template.lock` |
| `skcr list-targets` | List available bake targets |
| `skcr version` | Show version, commit, and build date |

## Bakefile format (`agentic.bake.yaml`)

```yaml
version: "1"

variables:
  project_name: SkillDemo
  owner_team: platform-engineering
  default_language: de
  governance_level: standard

skill_sources:
  output_dir: skills
  defaults:
    version: 0.1.0
    owner: platform-engineering
    license: MIT
    compatible_with:
      - codex
  skills:
    - name: gitlab-policy-reviewer
      description: Reviews GitLab security policies.
      compatible_with:
        - codex
        - gitlab-duo

    - name: documentation-reviewer
      description: Reviews documentation quality.
      compatible_with:
        - codex
        - claude-code

skills:
  source: agent-skills.lock
  mode: reference

targets:
  default:
    platforms:
      - codex
    render:
      skill_sources: true
      platform_files: true
      platform_skills: true
```

### `skill_sources` block

Defines canonical skill source directories that `skcr` should create under `skills/<name>/`.

- `output_dir` — root directory for skill sources (default: `skills`)
- `defaults` — default values merged into each skill definition
- `skills` — list of skill definitions

### `skills` block

Controls integration of installed/locked skills from `skpm` state.

- `source` — path to `agent-skills.lock` (default: `agent-skills.lock`)
- `mode` — `reference` | `copy` | `link` | `embed` (default: `reference`)
- `platforms` — limit skill integration to specific platforms

### `render` config (per target)

Controls what `skcr bake` generates:

- `skill_sources` — create canonical skill source skeletons (default: true when `skill_sources.skills` is non-empty)
- `platform_files` — render platform-specific instruction files (default: true)
- `platform_skills` — copy skill `SKILL.md` into platform directories (default: true when skill sources are configured)

## Workflows

### Create skill sources only

```bash
skcr init --target . --project-name SkillDemo --platform codex
# Edit agentic.bake.yaml — add entries to skill_sources.skills
skcr scaffold skills --target .
```

### Add skills via init flag

```bash
skcr init --target . --platform codex \
  --skill gitlab-policy-reviewer \
  --skill documentation-reviewer
skcr scaffold skills --target .
```

### Full project bake

```bash
skcr bake default --target . --write
```

### Package and publish with skpm

```bash
skpm validate skills/gitlab-policy-reviewer
skpm package skills/gitlab-policy-reviewer
skpm publish skills/gitlab-policy-reviewer --source myregistry
```

### Use installed skills from skpm

```bash
skpm add my-skill@^1.0.0
skpm lock && skpm install
skcr bake default --skills-from agent-skills.lock --write
```

## `skcr scaffold skill`

Creates one canonical skill skeleton from CLI flags.

```bash
skcr scaffold skill <name> [flags]
```

**Flags:**

| Flag | Default | Description |
| --- | --- | --- |
| `--output-dir` | `.` | Directory where the skill directory is created |
| `--version` | `0.1.0` | Initial semver version |
| `--description` | | Skill description for `skill.yaml` |
| `--owner` | | Skill owner |
| `--platform` | `claude-code`, `codex` | Compatible platform (repeatable) |
| `--license` | `MIT` | License |
| `--force` | | Overwrite existing files |
| `--dry-run` | | Preview without writing |

**Generated layout:**

```text
skills/<name>/
├── SKILL.md
├── skill.yaml
├── VERSION
├── CHANGELOG.md
├── README.md
├── LICENSE
└── tests/
    └── README.md
```

**After scaffolding:**

```text
Next steps:
  skpm validate skills/<name>
  skpm package skills/<name>
  skpm publish skills/<name> --source <registry>
```

## `skcr scaffold skills`

Creates all canonical skill skeletons declared in `agentic.bake.yaml`.

```bash
skcr scaffold skills [flags]
```

**Flags:**

| Flag | Default | Description |
| --- | --- | --- |
| `--target` | `.` | Target repository path |
| `--bakefile` | `<target>/agentic.bake.yaml` | Path to bakefile |
| `--dry-run` | | Preview without writing |
| `--force` | | Overwrite existing files |

Existing files are **preserved by default** — only missing files are created. Use `--force` to overwrite.

## `skcr bake`

Renders platform-specific files and optionally creates skill source skeletons.

```bash
skcr bake [target] [flags]
```

**Flags:**

| Flag | Default | Description |
| --- | --- | --- |
| `--target` | `.` | Target repository path |
| `--write` | | Write files to disk |
| `--plan` | | Show plan without writing |
| `--detailed-diff` | | Show full diffs in plan |
| `--platform` | | Render only one platform |
| `--skills-from` | | Read locked skills from a lock file |
| `--skills-mode` | | Override skill integration mode |

`skcr bake` does **not** call `skpm`, write `agent-skills.yaml` or `agent-skills.lock`, or talk to any registry.

## Validation

```bash
skcr validate --target .
skcr validate --platform codex
skcr validate --against-lock agent-skills.lock
skcr validate --skills
```

`skcr validate` checks:

- `agentic.bake.yaml` structure and platform names
- `skill_sources` names are valid (lowercase, hyphens, no duplicates)
- `skill_sources` platform names are valid
- Canonical skill source directories exist for configured skills
- Generated platform files match the lockfile state

After a clean validation, a hint is printed if `skill_sources` is configured:

```text
For package/lifecycle validation, run:
  skpm validate skills/<name>
```

## Boundary with `skpm`

`skcr` **may**:

- Create `agentic.bake.yaml`
- Create canonical skill source directories (`skills/<name>/`)
- Render platform-specific output files
- Copy / link / embed `SKILL.md` into platform paths
- Track generated files in `.agentic-template.lock`
- Validate generated project state

`skcr` **must not**:

- Bump skill versions
- Package or publish skills
- Resolve registry versions or download registry artifacts
- Write `agent-skills.yaml` or `agent-skills.lock`
- Install, update, or verify skills from registries
- Talk to SkillForge, Artifactory, GitLab, GitHub, or other registries

## Backwards compatibility

Existing `agentic.bake.yaml` files without a `skill_sources` block continue to work unchanged. The `skills` integration block for `agent-skills.lock` is unaffected.

## Clean

`skcr clean` removes only files tracked in `.agentic-template.lock` (generated platform outputs). It never deletes canonical skill sources under `skills/<name>/`.

```bash
skcr clean --plan
skcr clean --write
```
